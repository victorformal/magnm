## 3. PROBLEMAS CRÍTICOS IDENTIFICADOS

### 🔴 PROBLEMA 1: Variáveis de Ambiente do Servidor Não Configuradas

**Severidade:** 🔴 CRÍTICA

**Descrição:**
O servidor precisa de 2 variáveis de ambiente para enviar eventos via Conversions API (server-side tracking):

```
META_PIXEL_ID=1139772708143683
META_ACCESS_TOKEN=<YOUR_ACCESS_TOKEN>
```

**Impacto:**
- Eventos server-side NÃO são enviados para o Facebook
- O `sendPurchaseEvent()` em `app/api/meta/purchase-from-session/route.ts` falha silenciosamente
- A deduplicação de eventos não funciona corretamente

**Onde está o problema:**

**Arquivo:** `lib/meta/sendEvent.ts` (linhas 36-39)
```typescript
return {
  pixelId: process.env.META_PIXEL_ID || "1139772708143683",
  accessToken: process.env.META_ACCESS_TOKEN || "",  // ❌ VAZIO!
}
```

**O que está acontecendo:**
1. O código tenta ler `process.env.META_ACCESS_TOKEN`
2. Se não estiver configurado, usa uma string vazia `""`
3. Quando `_sendToPixel()` é chamado, joga um erro: `"Missing access token for pixel 1139772708143683"`
4. O erro é silenciado pelo `try/catch`

---

### 🔴 PROBLEMA 2: Sem Configuração de Multi-Pixel por Moeda

**Severidade:** 🟡 MÉDIA (se você quer rastrear 3 moedas com 3 pixels diferentes)

**Descrição:**
O código suporta 3 pixels por moeda (GBP, USD, EUR), mas nenhum está configurado:

```
META_PIXEL_ID_GBP=<YOUR_PIXEL_ID>
META_ACCESS_TOKEN_GBP=<YOUR_ACCESS_TOKEN>

META_PIXEL_ID_USD=<YOUR_PIXEL_ID>
META_ACCESS_TOKEN_USD=<YOUR_ACCESS_TOKEN>

META_PIXEL_ID_EUR=<YOUR_PIXEL_ID>
META_ACCESS_TOKEN_EUR=<YOUR_ACCESS_TOKEN>

NEXT_PUBLIC_META_PIXEL_ID_GBP=<YOUR_PIXEL_ID>
NEXT_PUBLIC_META_PIXEL_ID_USD=<YOUR_PIXEL_ID>
NEXT_PUBLIC_META_PIXEL_ID_EUR=<YOUR_PIXEL_ID>
```

**Impacto:**
- Se você quer usar **apenas 1 pixel** para todas as moedas: ✅ Funciona (com o Problema 1 resolvido)
- Se você quer usar **3 pixels diferentes**: ❌ Não vai funcionar sem configurar as vars

---

### 🟡 PROBLEMA 3: Fallback Implícito Causa Comportamento Inesperado

**Severidade:** 🟡 MÉDIA

**Descrição:**
O código usa fallbacks que podem causar confusão:

```typescript
const PIXEL_ID_PRIMARY = "1139772708143683"
const PIXEL_ID_GBP  = process.env.NEXT_PUBLIC_META_PIXEL_ID_GBP  || PIXEL_ID_PRIMARY
```

**O que acontece:**
1. Se `NEXT_PUBLIC_META_PIXEL_ID_GBP` não está definida, usa o pixel principal
2. No client-side, todos os 3 pixels (GBP, USD, EUR) se tornam `1139772708143683`
3. O `fbq('init', id)` é chamado 3 vezes com o mesmo ID (mas isso é OK, Facebook deduplicca)

**Impacto:**
- ✅ Se você quer 1 pixel: Funciona perfeitamente
- ⚠️ Se você quer 3 pixels: Confunde, porque todos se tornam 1

---

### 🟡 PROBLEMA 4: Validação Fraca no `_sendToPixel()`

**Severidade:** 🟡 MÉDIA

**Descrição:**

**Arquivo:** `lib/meta/sendEvent.ts` (linha 115-120)
```typescript
async function _sendToPixel(
  data: MetaEventData,
  pixel: PixelConfig,
): Promise<MetaApiResponse> {
  const { pixelId, accessToken } = pixel

  if (!accessToken) {
    throw new Error(`Missing access token for pixel ${pixelId}`)
  }
```

**Problema:**
- Se o `accessToken` está vazio, joga um erro
- O erro é silenciado pelo `try/catch` em `sendMetaEventForCurrency()`
- Nenhuma notificação ou log informativo é exibido ao desenvolvedor

**Impacto:**
- Os eventos são perdidos silenciosamente
- Difícil de debugar

## 7. PROBLEMAS COMUNS E SOLUÇÕES

### Problema: "Pixel não aparece em Events Manager"

**Causas Possíveis:**
1. ❌ Access Token não configurado (PROBLEMA 1)
2. ❌ Pixel ID errado
3. ❌ Meta tag de verificação de domínio não configurada
4. ❌ Navegador bloqueando rastreadores (Brave, Safari ITP)

**Solução:**
1. Verifique Vars em v0 → Verificar que `META_ACCESS_TOKEN` está preenchido
2. Verifique que `META_PIXEL_ID=1139772708143683`
3. Verifique que a meta tag de verificação está no `<head>`
4. Tente em um navegador diferente (Chrome)

---

### Problema: "Eventos client-side aparecem, mas não server-side"

**Causas Possíveis:**
1. ❌ `META_ACCESS_TOKEN` está vazio
2. ❌ O `sendPurchaseEvent()` está falhando silenciosamente
3. ❌ A rota `/api/meta/purchase-from-session` não está sendo chamada

**Solução:**
1. Adicione um console.log em `lib/meta/sendEvent.ts`:
```typescript
console.log("[v0] Sending to pixel:", pixel.pixelId, "token:", pixel.accessToken ? "✓" : "✗")
```
2. Verifique os logs do servidor: `vercel logs --tail`
3. Verifique se a rota de API está retornando erro

---

### Problema: "Eventos duplicados aparecem"

**Causas Possíveis:**
1. ✅ Comportamento esperado se client-side e server-side disparam
2. ❌ O `eventID` não é único
3. ❌ O mesmo evento é enviado múltiplas vezes

**Solução:**
- O Facebook automaticamente deduplicava eventos com o mesmo `eventID` dentro de 24h
- Se usar um `eventID` aleatório, cada evento é único (correto)
- Verifique que o `eventID` está sendo gerado corretamente
